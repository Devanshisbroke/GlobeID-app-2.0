# Sample Tokens & Payloads

## Kiosk Session Token (JWT-like)

Generated by `tokenService.createToken()` when a kiosk scans a passport.

```json
{
  "iss": "terracore",
  "kid": "kiosk-key-1",
  "iat": 1741305600,
  "exp": 1741305720,
  "session_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "kiosk_id": "kiosk-sim-001",
  "country_code": "IN",
  "nonce": "f8a3b2c1d4e5f678"
}
```

Encoded as 3-part base64url: `<header>.<payload>.<signature>`

## App QR Token (JWT-like)

Generated by `appGenerateQR()` when the user taps "Link at Kiosk."

```json
{
  "iss": "terracore",
  "kid": "app-key-1",
  "iat": 1741305610,
  "exp": 1741305640,
  "user_id": "usr-001",
  "nonce": "a1b2c3d4e5f67890"
}
```

TTL: 30 seconds. The nonce is consumed on first use (replay protection).

## Entry Receipt Token

Issued after successful verification. Long-lived (1 year).

```json
{
  "iss": "terracore",
  "kid": "backend-receipt-key",
  "session_id": "a1b2c3d4-...",
  "user_id": "usr-001",
  "country_code": "IN",
  "kiosk_id": "kiosk-sim-001"
}
```

## Flow Summary

1. Kiosk → `kioskVerifyPassport()` → session created (status: pending)
2. App → `appGenerateQR()` → QR token displayed
3. Kiosk → `kioskScanApp({ sessionId, appToken })` → verifies nonce, creates receipt
4. App receives `session:verified` event → plays welcome animation → shows receipt
